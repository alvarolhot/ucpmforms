ðŸ“˜ When to use API vs HTML + SDK vs Embedded
Use API when you need maximum control and/or server-side processing:
  • Server-side validation, enrichment, CRM/CDP writes
  • Custom UX + custom fields, multi-step journeys
  • When the browser should not call OneTrust directly
Use HTML + SDK (External Web Form) when you want a custom form but faster than building full API logic:
  • You keep your own form UI, but OneTrust SDK submits the receipt
  • Good for quick pilots where you still want a custom experience
  • Less backend work than API, more flexibility than Embedded
Use Embedded when you want speed and standard UX:
  • Fast deployment with OneTrust-provided form UI
  • Lower implementation effort
  • Great for demos / simple flows
Tip: Many customers start with Embedded (speed), move to External (custom UI), and later mature to API (advanced journeys + server-side governance).
